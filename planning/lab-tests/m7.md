# Lab-test evidence — M7: Multi-Agent Incident Crew

**Machine:** Apple Silicon (arm64), 16 GB · Rancher Desktop · native Ollama · ChromaDB 0.5.20
**Date:** 2026-07-05
**Crew:** `labs/m7/crew/` — four declarative profiles (Triage → Investigator → Fixer → Reviewer),
one shared `qwen2.5:1.5b`, `crew.py` glue. Reuses the Acme runbooks (agentic RAG) from M6.

## Approve path — incident WITH a runbook

```
INCIDENT: The checkout page is returning HTTP 503 errors for all users.
[TRIAGE]       AREA: web/checkout | SEV: SEV3 | Checkout service is down or misconfigured.
[INVESTIGATOR] ## Checkout 503 errors  (retrieved the exact runbook via RAG)
[FIXER]        kubectl scale deploy/web --replicas=5 -n prod   (quoted from the runbook)
[REVIEWER]     APPROVED: non-destructive, runbook-backed remediation ...
OUTCOME: APPROVED — ready for a human to apply
```

## Escalate path — incident with NO runbook (the review loop earns its keep)

Naive top-1 retrieval always returns a nearest chunk, so the crew adds a **relevance gate**: the
Investigator confirms the passage actually addresses the incident before the Fixer acts. Without it,
the crew confidently proposed the *payments* runbook for a *Kafka* outage — a real, dangerous failure
mode caught during validation.

```
INCIDENT: The Kafka event streaming cluster has stopped processing messages.
[TRIAGE]       Kafka | SEV3 | ...
[INVESTIGATOR] NO RUNBOOK FOUND — the knowledge base has no runbook for this incident. Escalate.
[FIXER]        I cannot propose a fix: no runbook covers this incident.
[REVIEWER]     REJECTED: no runbook-backed fix exists; escalate to a human on-call engineer.
OUTCOME: REJECTED — escalate
```

## CrewAI framework variant (documented, not the default)

Reuse repo `compose-for-agents/crew-ai` (CrewAI, `agents.yaml`/`tasks.yaml`). To run it anywhere it must
be **ported**: (1) delete the Compose `models:` block (Docker Model Runner) and point the agents'
`OPENAI_BASE_URL` at native Ollama's `/v1` (`http://host.docker.internal:11434/v1`, model `qwen2.5`);
(2) replace the `supergateway` MCP bridge with a **ToolHive**-managed MCP server (`thv run duckduckgo`).
CrewAI's role-based orchestration wants a more capable model than 1.5B (its default is `gemma3:4B`), so
this is shown as the *framework* option — reach for it when you need its structure; the declarative crew
above is the laptop-friendly default.
