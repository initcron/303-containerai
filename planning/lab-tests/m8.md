# Lab-test evidence — M8: Securing & Governing AI Workloads

**Machine:** Apple Silicon (arm64) · Rancher Desktop · **syft, trivy 0.72, grype, cosign 3.1** (brew)
**Date:** 2026-07-05
**Target:** `acme-support-agent:latest` (the M6 agent image, Debian 13 + Python 3.12).

## SBOM — Syft

```
$ syft acme-support-agent:latest -o spdx-json > sbom.spdx.json
   -> 96 packages, SPDX-2.3   (Debian debs + Python + binaries)
```

## Vulnerability scan — Trivy and Grype

```
$ trivy image --scanners vuln --severity CRITICAL,HIGH,MEDIUM acme-support-agent:latest
   Total: 64 (MEDIUM: 53, HIGH: 9, CRITICAL: 2)     e.g. gzip CVE-2026-41992 (HIGH), libacl1 CVE-2026-54369 (HIGH)
$ grype acme-support-agent:latest
   by severity: Critical 5, High 28, Medium 60, Low 7, Negligible 51
```

Two scanners, overlapping-but-different findings — the lesson: scanners disagree, so run more than one and
triage by fixable + severity. (Remediation = rebuild on a patched base / bump packages, then re-scan.)

## Sign & verify — Cosign (key-based, offline)

```
$ cosign generate-key-pair                      # cosign.key + cosign.pub (empty passphrase for the lab)
$ cosign sign   --key cosign.key <registry>/acme-support-agent:1.0.0
$ cosign verify --key cosign.pub <registry>/acme-support-agent:1.0.0
   -> "The signatures were verified against the specified public key"
```

(Validated against a local `registry:2`; GHCR works identically once you have a `write:packages` token.)

## Sandboxing generated / tool code — locked-down ephemeral container

```
$ docker run --rm --network none --read-only --cap-drop ALL --pids-limit 64 --memory 256m \
    python:3.12-slim python -c "print('sandboxed result:', sum(range(10)))"
   -> sandboxed result: 45
# and isolation is real — network egress is blocked:
$ docker run --rm --network none --cap-drop ALL python:3.12-slim \
    python -c "import urllib.request; urllib.request.urlopen('http://example.com', timeout=3)"
   -> URLError (network unreachable)
```

This is how the crew's Fixer would run any generated/tool code: no network, read-only rootfs, all
capabilities dropped, resource-capped, thrown away after one use (gVisor/ToolHive add deeper isolation).

## Verdict

✅ The open-source supply chain works end-to-end on a laptop: **Syft** SBOM → **Trivy/Grype** scan →
**Cosign** sign+verify, plus **sandboxed** code execution with proven network isolation. Wire these into
a GitHub Actions `build → scan → sign` pipeline (see `labs/m8/security-pipeline.yml`).
